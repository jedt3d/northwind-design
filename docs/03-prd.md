# Product Requirements Document (PRD) — Northwind Web App

> **Audience:** whole team · **Status:** outline (Phase 1, Cycle 1.2) · **Owner:** —

## 1. Product summary
A calm, clean, light-themed responsive web app for running Northwind Traders' daily operations in Thai, English, or Japanese.

## 2. Users & personas
- [ ] Sales (desktop-heavy), Purchasing, Warehouse (mobile-heavy), Manager, Admin

## 3. Feature list & user stories
For each feature: story, acceptance criteria, priority (MoSCoW), wireframe reference.

- [ ] F1 Login & language selection
- [ ] F2 Dashboard (my tasks: orders to process, POs to approve, low stock)
- [ ] F3 Companies directory & detail
- [ ] F4 Products & categories
- [ ] F5 Order management (list, create wizard, detail, status actions)
- [ ] F6 Purchase orders (list, create, approve, receive)
- [ ] F7 Inventory (on-hand, transactions, adjustment)
- [ ] F8 Reports
- [ ] F9 Admin (employees, roles)

## 4. Out of scope (v1)
- [ ] Dark theme, offline mode, external accounting integration, multi-warehouse

## 5. Release plan
- v0.1-design → v0.2-backend → v1.0 (see PLAN.md cycles)

## 6. Metrics of success
- [ ] Core flows completable on 360px mobile; all strings translated in 3 languages
